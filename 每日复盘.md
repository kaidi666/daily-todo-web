**PRD**
**产品名**
`每日复盘`

**一句话定位**
一款面向手机优先场景的轻量 Web 应用，帮助用户随手记录当天做过的事，并在晚上快速完成每日复盘。

**目标用户**
经常想不起来自己一天做了什么、希望用最小成本养成记录习惯的人。
当前主要用户就是你自己，所以第一版优先考虑：
- 手机上快速输入
- 晚上回顾方便
- 数据可导出，不被锁定

**核心问题**
- 白天发生了很多碎片小事，但事后很难回忆
- 记录工具太重时，会懒得打开或懒得整理
- 晚上复盘时，缺少结构化提示，不知道该怎么写

**产品目标**
- 让用户能在手机上 3 秒内记下一条事情
- 晚上打开后，能快速完成简短复盘
- 能方便查看过去某一天做了什么
- 能把数据导出成 Markdown，保留可迁移性

**第一版范围**
包含：
- 今日记录
- 今晚复盘
- 历史查看
- Markdown 导入导出
- 本地自动保存
- 手机端优先 UI

不包含：
- 登录注册
- 云同步
- 多设备实时同步
- 标签系统
- 统计图表
- 复杂日历视图
- 提醒通知

**核心功能**
1. 今日记录
- 顶部显示今天日期
- 提供一个快速输入框
- 用户输入内容后，点击“添加”或按回车加入今日清单
- 每条事项支持编辑、删除、标记完成
- 每条事项自动记录创建时间
- 所有修改自动保存

2. 今晚复盘
- 放在今日页下半部分，不单独拆页
- 提供三个复盘字段：
  `今天完成了什么`
  `今天遇到了什么问题`
  `明天打算做什么`
- 支持多行输入
- 输入过程中自动保存

3. 历史查看
- 默认按日期倒序展示历史记录
- 每个日期卡片显示：
  日期、事项数、已完成数、是否已写复盘
- 点击进入某天详情页
- 支持按月份筛选
- 第一版不强制做月历，只做列表更适合手机

4. 导入导出
- 导出全部记录为单个 Markdown 文件
- 导入符合模板的 Markdown 文件恢复数据
- 导入时覆盖同日期已有记录，或进行合并
- 第一版建议采用“同日期覆盖”规则，逻辑更简单

**关键设计原则**
- 移动端优先
- 输入成本低于管理成本
- 自动保存优先于手动保存
- 查看历史要快于编辑历史
- 数据格式清晰稳定，便于导出和迁移

**信息架构**
底部导航建议 3 个入口：
- `今日`
- `历史`
- `设置`

页面如下：
- 今日页
- 历史列表页
- 历史详情页
- 设置页

**页面流程**
1. 今日页流程
- 打开应用
- 默认进入今天页面
- 页面顶部显示日期，如 `2026-04-12 周日`
- 底部或顶部有快速输入框
- 输入“开会”后点击添加
- 新事项立即出现在列表里，并显示时间
- 用户可继续添加多条
- 晚上滑到下方填写复盘内容
- 所有内容自动保存

2. 历史页流程
- 点击底部 `历史`
- 默认看到按天倒序的记录列表
- 可切换月份筛选，如 `2026-04`
- 点击某一天进入详情
- 查看当天事项和复盘内容
- 可在详情页继续编辑

3. 设置页流程
- 点击底部 `设置`
- 可进行：
  导出 Markdown
  导入 Markdown
  清空本地数据
  查看版本信息

**页面说明**
**1. 今日页**
模块：
- 日期头部
- 快速输入区
- 今日事项列表
- 今晚复盘区

字段：
- 日期展示
- 输入框 placeholder：`今天做了什么？`
- 添加按钮
- 事项项字段：文本、时间、完成状态
- 复盘输入框三项

交互：
- 输入框聚焦后弹起手机键盘
- 回车直接添加
- 添加后清空输入框
- 编辑事项时支持页内直接修改
- 删除操作建议二次确认或提供撤销
- 复盘区实时自动保存

**2. 历史列表页**
模块：
- 月份筛选
- 日期记录列表

每条记录展示：
- 日期
- 星期
- 事项总数
- 完成数
- 是否已写复盘

交互：
- 点击卡片进入详情
- 向下滚动加载历史
- 月份筛选后只显示该月

**3. 历史详情页**
模块：
- 日期头部
- 当天事项列表
- 复盘内容区

交互：
- 支持查看和编辑
- 返回历史页时保持原筛选位置

**4. 设置页**
模块：
- 导出数据
- 导入数据
- 清空数据
- 关于应用

交互：
- 导出后下载 `.md`
- 导入前提示数据覆盖风险
- 清空前必须二次确认

**数据结构**
建议第一版按天存储：

```ts
type RecordItem = {
  id: string;
  text: string;
  createdAt: string;
  completed: boolean;
};

type DayRecord = {
  date: string; // YYYY-MM-DD
  items: RecordItem[];
  summary: string;
  problems: string;
  tomorrow: string;
  updatedAt: string;
};

type AppData = {
  version: string;
  days: Record<string, DayRecord>;
};
```

这样做的好处：
- 按日期查当天记录很方便
- 导出 Markdown 简单
- 历史列表生成简单
- 后续迁移到云端也容易

**存储方案**
第一版建议：
- 优先 `localStorage`，实现快
- 如果后续记录量变大，再迁移 `IndexedDB`

建议规则：
- 每次新增、编辑、删除都立即持久化
- 页面初始化时自动读取本地数据
- 如果当天没有记录，自动创建空白当天数据结构

**Markdown 格式规范**
建议固定模板：

```md
# 2026-04-12

## 事项
- [ ] 开会
- [x] 写日报
- [ ] 买菜

## 今天完成了什么
- 和客户把需求确认清楚

## 今天遇到了什么问题
- 下午被杂事打断很多次

## 明天打算做什么
- 先完成首页原型
```

解析规则建议：
- `# 日期` 表示一天记录开始
- `## 事项` 下读取任务列表
- `- [ ]` 与 `- [x]` 对应完成状态
- 其余三个 `##` 标题分别映射三个复盘字段

**非功能需求**
- 手机端适配良好，宽度优先考虑 `360px-430px`
- 输入和切页操作流畅
- 自动保存可靠
- 不依赖后端即可运行
- 支持安装为 PWA 更佳，但可放到第二阶段

**成功标准**
第一版上线后，如果满足以下几点，就算成功：
- 你能连续使用 7 天
- 白天愿意随手打开记录
- 晚上能通过历史和事项回想起当天内容
- 导出 Markdown 后内容清晰可读

**后续迭代方向**
可以放到第二版：
- 日历视图
- 搜索关键词
- 标签或分类
- 每周复盘
- 数据统计
- 云同步
- 提醒通知

**页面流程图简版**
```text
启动应用
  -> 今日页
    -> 添加事项
    -> 编辑/删除/完成事项
    -> 填写今晚复盘
  -> 历史页
    -> 选择月份
    -> 查看日期列表
    -> 进入某天详情
  -> 设置页
    -> 导出 Markdown
    -> 导入 Markdown
    -> 清空数据
```
