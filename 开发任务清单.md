**开发任务清单**

下面这版按 `MVP` 的实际开发顺序拆分，适合直接进入前端实现。默认技术前提是：`移动端优先 Web App`、`本地存储`、`无后端`。

**一、项目初始化**
1. 创建前端项目脚手架
- 选择框架：`Vue` 或 `React`
- 配置基础路由
- 配置移动端适配与全局样式重置
- 建立基础目录结构：`pages`、`components`、`stores`、`utils`、`types`

验收标准：
- 项目可本地运行
- 手机尺寸下页面无明显样式错乱
- 已有空白的 `今日 / 历史 / 设置` 页面路由

2. 建立全局样式系统
- 定义颜色、字号、间距、圆角、阴影等设计变量
- 统一按钮、输入框、卡片、列表项的基础样式
- 处理安全区和底部导航区域

验收标准：
- 页面在手机端可正常阅读和点击
- 基础组件风格统一
- 底部导航不会遮挡内容

**二、数据模型与本地存储**
3. 定义数据类型
- 定义 `RecordItem`
- 定义 `DayRecord`
- 定义 `AppData`
- 统一日期格式与时间格式工具函数

验收标准：
- 类型定义完整
- 所有页面都基于统一数据结构开发

4. 封装本地存储模块
- 读取应用数据
- 写入应用数据
- 初始化空数据结构
- 处理版本号字段
- 提供获取某天记录、更新某天记录的方法

验收标准：
- 刷新页面后数据不丢失
- 首次打开能自动初始化
- 任意数据修改后能正确持久化

5. 建立状态管理
- 管理当前日期
- 管理当天记录
- 管理历史记录列表
- 提供新增事项、编辑事项、删除事项、更新复盘字段等 action

验收标准：
- 页面间数据一致
- 操作后 UI 实时更新
- 刷新后数据仍正确恢复

**三、核心页面开发**
6. 开发底部导航框架
- 实现 `今日 / 历史 / 设置` 底部导航
- 当前页面高亮
- 适配手机单手点击区域

验收标准：
- 三个页面之间切换流畅
- 移动端点击区域足够大
- 导航固定且不挡主要内容

7. 开发今日页基础结构
- 显示当天日期与星期
- 布局快速输入区
- 布局事项列表区
- 布局复盘输入区

验收标准：
- 今日页在手机上信息层次清晰
- 输入区、列表区、复盘区完整展示

**四、今日记录功能**
8. 实现快速添加事项
- 输入框输入事项文本
- 点击添加或按回车新增事项
- 自动记录创建时间
- 新增后清空输入框
- 空内容禁止提交

验收标准：
- 可连续快速添加多条事项
- 每条事项都带创建时间
- 空白输入不会生成脏数据

9. 实现事项列表展示
- 展示事项文本
- 展示时间
- 展示完成状态
- 空列表时展示引导文案

验收标准：
- 列表展示清晰
- 时间格式统一
- 当天无事项时页面不空白

10. 实现事项完成状态切换
- 点击勾选切换完成/未完成
- 已完成事项样式变化

验收标准：
- 状态切换后立即保存
- 刷新后完成状态正确保留

11. 实现事项编辑
- 支持页内编辑事项文本
- 编辑完成后自动保存
- 可取消编辑

验收标准：
- 编辑流程顺畅
- 编辑后列表即时更新
- 不会误删原内容

12. 实现事项删除
- 点击删除按钮删除事项
- 删除前可选确认机制，或提供短暂撤销

验收标准：
- 删除后数据立即更新
- 不会误触频繁发生
- 刷新后删除结果正确

**五、复盘功能**
13. 实现三个复盘输入区
- `今天完成了什么`
- `今天遇到了什么问题`
- `明天打算做什么`

验收标准：
- 三个字段均支持多行输入
- 文本内容可单独保存

14. 实现复盘自动保存
- 输入过程中自动保存
- 页面刷新后内容恢复
- 字段为空时也能正确处理

验收标准：
- 用户无需手动点击保存
- 输入后刷新页面不丢失内容

**六、历史功能**
15. 开发历史列表页
- 按日期倒序显示历史记录
- 每项展示日期、星期、事项总数、完成数、是否写了复盘
- 点击进入详情页

验收标准：
- 历史数据排序正确
- 卡片信息清晰
- 能稳定跳转到详情页

16. 实现月份筛选
- 提供月份切换或月份选择器
- 仅显示该月记录

验收标准：
- 月份切换后数据正确过滤
- 无记录月份能正常展示空态

17. 开发历史详情页
- 显示当天所有事项
- 显示三个复盘字段
- 支持编辑当天内容

验收标准：
- 能查看任意日期详情
- 修改后返回历史页数据同步正确

**七、导入导出功能**
18. 实现 Markdown 导出
- 将所有日期记录组装成固定 Markdown 格式
- 触发下载 `.md` 文件
- 确保导出顺序稳定

验收标准：
- 导出文件可直接打开阅读
- 事项与复盘内容格式正确
- 多天记录能完整导出

19. 实现 Markdown 导入
- 上传 Markdown 文件
- 解析日期、事项、完成状态和复盘内容
- 写入本地数据
- 处理重复日期覆盖逻辑

验收标准：
- 符合模板的 Markdown 能成功导入
- 导入后历史页可立即看到数据
- 同日期覆盖逻辑稳定

20. 导入异常处理
- 文件格式错误提示
- 缺少字段时合理兜底
- 导入前给出覆盖提示

验收标准：
- 格式异常不会导致应用崩溃
- 用户能理解导入失败原因

**八、设置页**
21. 开发设置页基础功能
- 导出按钮
- 导入按钮
- 清空本地数据按钮
- 版本信息展示

验收标准：
- 所有入口清晰可见
- 设置页操作反馈明确

22. 实现清空数据
- 二次确认
- 清空后重建空数据结构
- 返回今日页时不报错

验收标准：
- 清空操作安全
- 清空后应用状态正常

**九、移动端体验优化**
23. 优化输入体验
- 输入框聚焦时页面不抖动
- 键盘弹出后主要操作仍可见
- 按钮尺寸适合手指点击

验收标准：
- 主流手机尺寸上输入顺畅
- 没有明显键盘遮挡问题

24. 优化滚动与布局
- 长列表滚动流畅
- 底部导航不遮挡输入区
- 复盘区域在小屏手机上仍可操作

验收标准：
- 主要交互可单手完成
- 视觉层次清楚

**十、异常与边界处理**
25. 处理空状态
- 今日无事项空状态
- 历史无记录空状态
- 月份无数据空状态

验收标准：
- 页面不会出现“空白但无提示”的情况

26. 处理边界数据
- 超长文本
- 同一天大量事项
- 导入空文件或非法文件
- 本地数据损坏时自动兜底恢复

验收标准：
- 应用在异常情况下不崩溃
- 用户能看到明确提示

**十一、测试与验收**
27. 功能联调测试
- 新增、编辑、删除、完成事项
- 复盘自动保存
- 历史查看与编辑
- 导入导出闭环测试

验收标准：
- 完整核心流程跑通
- 刷新后数据一致

28. 移动端验收测试
- 模拟常见手机尺寸
- 测试输入、切页、滚动、导入导出流程
- 检查触控区域和可读性

验收标准：
- 手机上能连续完成一天记录和复盘流程
- 无明显阻塞操作

**推荐开发优先级**
`P0`
- 项目初始化
- 数据结构与本地存储
- 今日页
- 新增/编辑/删除/完成事项
- 复盘输入与自动保存

`P1`
- 历史列表页
- 历史详情页
- 月份筛选

`P2`
- Markdown 导出
- Markdown 导入
- 设置页清空数据

`P3`
- 细节交互优化
- 空状态优化
- 异常处理完善

**建议里程碑**
1. 第一阶段：先做“今天能记”
- 完成今日页
- 完成本地保存
- 完成复盘输入

2. 第二阶段：再做“过去能看”
- 完成历史列表
- 完成历史详情
- 完成月份筛选

3. 第三阶段：最后做“数据能带走”
- 完成导出
- 完成导入
- 完成设置页和清空数据

**给你的建议**
如果你是自己开发，我建议你先只盯住一个最小闭环：

- 今日页
- 本地存储
- 事项 CRUD
- 三个复盘字段自动保存

这四块做完，你其实已经有一个能开始每天使用的版本了。后面的历史、导入导出，都可以在你真实用了几天之后再补，会更贴合习惯。
